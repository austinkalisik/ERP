1. Daily Start Guide in VS Code

Use this section every day when you forget how to start the CRM.

  1. Open VS Code.
  2. Open folder Z:\NextGen Projects\NextGen CRM.
  3. Open the terminal with Ctrl + `.
  4. If the terminal is not in the project folder, run:
cd "Z:\NextGen Projects\NextGen CRM"

Recommended daily development command:

composer run dev

This starts Laravel, the Laravel queue listener, and Vite frontend live reload.

Then open a second VS Code terminal and run Caddy:

caddy run --config Caddyfile

Open:

https://192.168.88.133:8443/login

The browser may show a certificate warning because this project uses Caddy internal TLS for local HTTPS. That is normal for local/internal development.

Alternative Manual Start

If composer run dev does not work, use three terminals.

php artisan serve --host=127.0.0.1 --port=8000
npm run dev -- --host 127.0.0.1
caddy run --config Caddyfile

3. Database You Are Using

The project uses MySQL. The current non-secret database settings are:

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE="nextgen crm"
DB_USERNAME=root

Laravel sessions, queues, and cache are also configured to use database tables:

SESSION_DRIVER=database
QUEUE_CONNECTION=database
CACHE_STORE=database

Useful database commands:

php artisan migrate
php artisan db:seed
php artisan migrate:fresh --seed

Only use migrate:fresh --seed on local development data because it deletes and recreates tables.

14. Expert Notes

  • Keep one source of truth for ports. The current Caddyfile expects Laravel on port 8000. Some older notes mention 8001.
  • composer run dev is the best development start command because it starts Laravel, queue, and Vite together.
  • Caddy should stay in a separate terminal because it is the reverse proxy.
  • Do not expose .env passwords in Git or documentation.
  • Do not use php artisan migrate:fresh --seed unless you are comfortable deleting local data.
  • Before production launch, add backups, audit logs, real SMTP, strong passwords, scheduler, queue service, and HTTPS with a real domain.
