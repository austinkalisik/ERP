Production readiness, administrator guide, and hardware integration plan for MikroTik or UniFi access points.

Prepared For National Airports Corporation
System NAC Smart WiFi Captive Portal
Prepared Date 20 May 2026
Current Mode MikroTik HotSpot handoff, UniFi ready

1. Executive Summary

The NAC WiFi Captive Portal is a guest WiFi access system for visitors, passengers, staff guests, and other approved users. It provides a branded splash page, guest login, free advertisement-supported access, paid access packages, live session tracking, advertisement management, subscription plans, and controller settings for real captive portal deployment.

The system is designed so that when MikroTik or UniFi access point hardware arrives, the administrator can configure the portal URL, controller settings, SSIDs, access time, and HotSpot bridge file from the Admin Console. The goal is to make deployment practical without rewriting code.

Plain explanation: A user connects to NAC WiFi, the router or access point sends them to this portal, the user signs in or watches an advert or pays, and the router/controller then allows that device to access the internet for the approved time.

3. System Purpose

The portal exists to control and record WiFi access. It is not just a web page. It works together with network hardware such as MikroTik RouterOS HotSpot or UniFi external portal features. The hardware blocks users before login. The portal collects the user details and chooses whether the user should receive free or paid access.

AreaPurpose
Open PortalPublic splash page for users who connect to the WiFi network.
Guest LoginCollects name, organization, phone or email before access is granted.
Free AccessShows an active advertisement first, then creates an access grant.
Paid AccessShows paid packages, records the payment path, and creates premium access after approval.
Admin ConsoleLets staff manage sessions, ads, plans, settings, logo, and controller readiness.
Controller IntegrationConnects the business flow to MikroTik or UniFi so the user actually receives internet access.

4. Basic User Flow

1. ConnectUser joins the NAC WiFi SSID from phone, laptop, or tablet.
2. RedirectMikroTik or UniFi sends the user to the captive portal splash page.
3. RegisterUser enters name and phone or email. The system records IP and device details where available.
4. AccessUser watches an advert for free access or selects a paid package. The router/controller authorizes internet access.

8. Advertisements and Free Access

Free access is controlled through active advertisements. When the user chooses free access, the system selects one active advertisement and shows it for a short countdown. After the countdown, the system records the advert view and grants free access for the configured time.

Figure 5: Advertisement Manager for upload, edit, activate, deactivate, and delete.

How Free Ads Work

  1. Administrator uploads an image or video advert.
  2. Administrator marks the advert active.
  3. Guest selects Free Access.
  4. Portal displays the advert and countdown.
  5. System records the view and click if the user opens the advertiser link.
  6. Portal creates an access grant and hands the user back to the router/controller.

For production captive portal use, uploaded local media is recommended because unauthenticated WiFi clients may not be allowed to load external images or videos before internet access is granted.

9. Plans, Paid Access, and Payments

Plans define paid WiFi packages such as 1 Hour, 3 Hours, or 1 Day. Each plan has a duration, price, active/off status, and sort order. The user sees only active plans on the paid access page.

Figure 6: Plans page where paid access packages can be edited, activated, or deleted.

How Paid Access Works

  1. Administrator creates at least one active plan.
  2. Guest selects Paid Subscription.
  3. Guest chooses a package.
  4. Payment is processed through configured provider or manual approval during testing.
  5. After approval, the system creates a premium access grant for the selected duration.
Production payment rule: Live paid access should only be granted after verified payment callback, webhook, or approved manual payment confirmation.

12. MikroTik Captive Portal Integration

MikroTik RouterOS HotSpot is the current recommended direct integration path. RouterOS controls internet access. The PHP portal handles branding, guest registration, ads, payment flow, and session records.

Required MikroTik Items

  • MikroTik router or CHR with HotSpot configured.
  • WiFi AP or bridge connected to the HotSpot network.
  • A shared HotSpot user that the portal can post back to RouterOS after approval.
  • The portal server IP reachable by HotSpot clients.
  • The generated hotspot/login.html uploaded to RouterOS HotSpot files.

MikroTik Deployment Steps

  1. Configure the HotSpot network on MikroTik.
  2. Create the shared HotSpot username/password.
  3. In Admin Settings, select MikroTik HotSpot handoff.
  4. Set Public Portal Host/IP and Portal Path.
  5. Enter the shared HotSpot username/password.
  6. Open System Control and regenerate the HotSpot bridge file.
  7. Upload hotspot/login.html to the MikroTik HotSpot file area.
  8. Connect a test client to WiFi and open http://neverssl.com.
  9. Complete guest login, free advert, or paid flow.
  10. Confirm the client appears in MikroTik active HotSpot users and has internet access.
Result: The user is not just shown a success page. RouterOS is used to authorize real network access.

13. UniFi Captive Portal Integration

UniFi integration depends on the UniFi Network version, gateway/AP model, and available external portal/API method. The system is ready to hold UniFi controller settings and route the guest flow through the portal. Final authorization must be connected to the confirmed UniFi method when hardware arrives.

Required UniFi Items

  • UniFi access point or UniFi gateway/access point setup.
  • UniFi Network application/controller reachable from the portal server.
  • Guest WiFi network with captive portal or external portal enabled.
  • External portal URL pointing to the NAC portal.
  • UniFi site ID and API access method confirmed for the installed version.

UniFi Deployment Steps

  1. Adopt the UniFi AP into the controller.
  2. Create the guest WiFi SSID.
  3. Enable guest/captive portal or external portal mode.
  4. Set the external portal URL to the NAC portal base URL.
  5. In Admin Settings, choose UniFi external portal or Spotipo/UniFi depending on the final controller method.
  6. Enter controller URL, API key or token, and site ID.
  7. Connect a test client and confirm the redirect opens the NAC portal.
  8. Complete free or paid access and confirm the client is authorized by UniFi.
Important: UniFi final authorization must be tested with the real purchased hardware and controller version. Do not mark enforcement as integrated until a real WiFi client receives internet through UniFi after portal approval.

15. Access Types Explained

Free Advertisement Access

Free access is used when NAC or a sponsor wants to provide limited WiFi in exchange for advert exposure. The user must watch or interact with an advert before access is granted. The system records advert views and clicks.

Paid Access

Paid access is used when users need longer or premium connectivity. The administrator controls the plan name, price, and duration. Payments should be verified before final access is granted.

Manual or Testing Access

During office testing, the system can record sessions without final router enforcement. This is useful for checking screens and reports, but production access must be tested through MikroTik or UniFi.

17. Final Recommendation

The system is ready as a captive portal application layer and administrator console. The remaining production proof depends on the arrival and configuration of MikroTik or UniFi hardware. Once the AP/router is installed, the most important test is a real client test from the guest WiFi network.

Go-live rule: Do not consider the system live until a phone or laptop connects to the guest WiFi, is redirected to the NAC portal, completes the flow, appears authorized on the router/controller, and receives internet access.

Management Summary

This portal gives NAC a controlled, branded, reportable WiFi access system. It supports free sponsored advertising access, paid package access, administrator reporting, and a path to MikroTik or UniFi captive portal enforcement. The system is structured so non-technical staff can manage day-to-day content while IT controls the router/controller integration.
